# AI-School-Recommendation-System

# 项目背景与目标

## 背景说明

### 市场概述

1. 多方研究显示，出国留学需求在疫情后持续恢复，中国依然是全球最大留学生源国之一；智库 CCG 连续蓝皮书梳理了恢复态势与目的地多元化（除美英澳加外，欧陆与东南亚热度上升）。
2. 市场增速：第三方行业研究给出 2024–2029 年复合增速 ~16–17% 的预测（方法与口径各异，供参考，最终以你自研口径为准）
3. 2024–2025 关键政策与目的地变化:
    1. **英国:** 继续把英国作为“性价比+路径清晰”的目的地重点;
    2. **澳大利亚**：2024 年起多项收紧（更高英语门槛、GTE→GS、限制在境内「签证跳岛」、部分签证费用上调、毕业工签时长调整）。
    3. **加拿大:** 自 2024 年推行**学签配额**，2025 年继续实施并细化省配额，留学生总量较 2023 年显著下降。
    4. **美国:** F-1 签证指引与流程工具持续更新（SEVIS/DS-160 与预约工具等）；各地面签等待周期波动较大。

### 目标用户与痛点（半 DIY 人群）

1. **目标用户：**预算敏感、愿意自己做功课、希望获得流程掌控感与透明报价的本科/授课型硕士申请者与家长。
2. **核心痛点：**
    1. 信息分散、规则多变（目的地与签证政策频繁调整）;
    2. 传统机构价格高、打包严（文书/选校/投递一揽子），难以**按需购买;**
    3. 进度不可视 & 沟通成本高（顾问人力窗口期有限）;
    4. 文书与材料质量把控难（担忧「AI 痕迹」「查重」与合规）;
    5. 录取≠落地：从签证、住宿到注册报到、开户、电话卡、医保等“最后一公里”繁琐。

### 竞品地图（中国市场相关）

A. 传统留学机构（高客单 + 全托管）

新东方前途出国、启德教育、金吉列、IDP 等，提供一站式方案与顾问服务，品牌背书强、线下网点多，但费用较高、透明度与可组合性有限。

B. 平台型/互联网留学（强调流程化与匹配）

UI丑，在小红书上的曝光率差。

### 机会空间（AI 留学全流程伴侣）

1. “国家/院校/签证规则”实时引擎
    1. 把英国 Graduate Route、澳洲 GS/英语门槛、加拿大配额与省份要求等，建模为**可执行规则**，驱动选校可行性、签证策略与时间线建议。
2. 半 DIY 的“按需购买”产品化
    1. 核心模块可拆分：①选校诊断与清单 ②材料收集与校验 ③文书工作台（故事库→多版本→AI 痕迹治理→人工审核）④多院校批量投递编排 ⑤签证资料管家与面签模拟 ⑥落地清单（住宿/入学注册/开户/电话卡/保险/IT 账户）与**到校倒排日历**。
3. 可信与合规
    1. 成**官方来源**的数据接口或定期爬虫校验；在文书侧提供**原创性证明**（编辑轨迹、素材来源、相似度报告）与**招生官友好模式**（减少模板化语气/避免虚假堆料）。参考目的地官方页面作为“合规锚”。

### 产品目标

AIvy不是一个留学平台，而是属于你的AI留学伴侣。从萌生留学想法到落地梦校，AIvy全程都会为你提供专业且个性化的辅导与支持。

# 三、用户与场景分析

## 目标用户

### 1. 主用户：大学生（本科/研究生申请者）

- **年龄**：18～26 岁
- **身份/阶段**：
    - 大三/大四在校生（准备申请硕士）
    - 已毕业 1–2 年的职场新人（申请海外硕士）
- **地区分布**：一二线城市（北京、上海、广州、成都、杭州），以及部分高校集中的省会
- **学习背景**：重点高校/二本及以上，专业集中在商科、计算机、设计、传媒等热门方向
- **收入与消费能力**：预算有限（希望控制在传统机构 20–50% 成本内），习惯分阶段、分功能付费
- **使用习惯**：
    - 每天使用手机 4–6 小时，偏好微信、小红书、知乎、B站获取留学信息
    - 喜欢自助操作、信息透明
    - 对 AI 工具有较高接受度（熟悉 ChatGPT、Grammarly、Notion AI 等）
- **核心痛点**：
    1. 不信任“高价机构”但又缺乏系统化指导
    2. 留学流程复杂（选校、材料、投递、签证、落地），信息分散
    3. 文书难写，既怕 AI 痕迹太重，又怕“模板化”被招生官识破
    4. 想自己主导申请节奏，但需要智能提醒和结构化支持

### 2. 次用户：学生家长（陪伴与决策支持者）

- **年龄**：40～55 岁
- **职业**：企事业单位职员、中小企业主、教师、医生等
- **使用场景**：更多通过微信小程序或网页端查看子女申请进度
- **痛点**：
    1. 担心被骗、不懂流程、看不懂英文文件
    2. 想要**进度透明**、**费用清晰**
    3. 希望有“智能监控”与“官方核验”功能
    4. 需要信任感与陪伴感，而非纯技术界面

### 3. 潜在专业用户（学校辅导员 / 校园代理）

- **角色**：帮助学生完成推荐信或资料收集的中介节点
- **动机**：提升工作效率、减少重复文书审阅
- **需求**：批量导入学生资料、一键生成模板、AI 检测原创度

## 典型用户使用场景

| 场景 | 描述 | 用户需求 |
| --- | --- | --- |
| **1. 晚上宿舍内** | 学生用手机输入 GPA、专业、预算、目标国家，AI 自动生成选校清单与录取概率图表。 | 想快速获得**可行性报告**与申请方向指导。 |
| **2. 地铁上/午休时间** | 学生在 App 查看“申请进度仪表盘”：哪些材料已提交、签证预约进度、学校录取状态。 | 想**随时掌握进度**、减少焦虑。 |
| **3. 家长端在微信小程序** | 家长打开“申请追踪视图”，看到孩子的申请节点（材料递交、面签通过、住宿确认等）。 | 想获得**安全感与透明度**。 |
| **4. 晚上文书写作时** | 学生打开“AI 文书工作台”，导入履历，AI 生成 PS/Essay 草稿，标出修改建议与AI痕迹比例。 | 想得到**灵感+安全的文书辅助**。 |
| **5. 临近签证阶段** | 学生收到系统推送：“你申请的国家（澳大利亚）签证规则更新，英语门槛提升，请检查材料”。 | 需要**实时政策提醒与风险预警**。 |
| **6. 到达国外后** | 学生使用“落地清单”功能：查看开户、电话卡、注册、体检、宿舍入住等待办事项。 | 想**全流程闭环**，减少信息焦虑。 |


## 详细功能描述与说明

1️⃣ 用户登录与档案建立

| 项目 | 说明 |
| --- | --- |
| **功能名称** | 用户登录注册 / 档案创建 |
| **功能描述** | 用户可通过手机号或邮箱注册账号，填写个人学术档案与留学偏好。 |
| **用户流程** | 首页→点击“开始使用”→输入手机号/邮箱→填写个人信息表单→提交→进入推荐页 |
| **输入** | 姓名、邮箱/手机号、最高学历、GPA、语言成绩、预算、目标国家/专业方向 |
| **输出** | 创建完成的“个人档案卡” + 推荐入口按钮 |
| **状态变化** | 加载中→成功→跳转至“选校推荐” |
| **交互逻辑** | 档案数据将同步传入“选校推荐模块”作为输入 |
| **优先级** | 🟡 **P2（非紧急）** |
|  |  |

2️⃣ **智能选校推荐模块**

| 项目 | 说明 |
| --- | --- |
| **功能名称** | AI 智能选校推荐 |
| **功能描述** | 基于用户档案生成匹配度报告，展示目标国家下最合适的大学与专业，并提供录取概率与申请建议。 |
| **用户流程** | 档案建立 → 点击“AI 生成推荐” → 系统计算 → 输出结果卡片 |
| **输入** | GPA、语言成绩、现专业、目标专业，预算、目标国家、偏好 |
| **输出** | 院校列表（匹配度%）+ “建议说明” + “查看详情”按钮 |
| **状态变化** | 加载中（动画）→ 推荐结果展示 → 用户可收藏/调整参数重算 |
| **交互逻辑** | 与“进度追踪模块”联动，用户选择某院校后自动添加到申请清单 |
| **优先级** | 🟢 **P0（必须实现！！）** |

3️⃣ AI 文书生成模块

| 项目 | 说明 |
| --- | --- |
| **功能名称** | 文书生成与润色（PS/Essay） |
| **功能描述** | 用户输入个人经历，AI 自动生成 Personal Statement 草稿，可查看修改建议与AI痕迹比率。 |
| **用户流程** | 选校完成 → 点击“生成文书” → 填写经历要点 → AI 生成初稿 → 用户编辑与保存 |
| **输入** | 用户输入的经历关键词、项目、成就描述 |
| **输出** | AI 生成文书草稿（支持复制 / 下载 / 评分） |
| **状态变化** | 生成中→展示结果→用户修改→保存成功 |
| **交互逻辑** | 与“选校模块”共享用户信息（自动填充学校名与专业） |
| **优先级** | 🟢 **P1（重要但可延期）** |

4️⃣ 留学进度追踪模块

| 项目 | 说明 |
| --- | --- |
| **功能名称** | 留学申请进度仪表盘 |
| **功能描述** | 可视化展示申请流程（选校→文书→网申→签证→入学），用户可手动或自动标记节点状态。 |
| **用户流程** | 用户登录 → 进入“进度”页 → 查看甘特图或时间轴 → 点击节点更新状态 |
| **输入** | 节点状态、预计完成日期 |
| **输出** | 可视化时间线 + 节点提示（完成/待办/逾期） |
| **状态变化** | 状态更新后仪表盘刷新，触发提醒通知 |
| **交互逻辑** | 与政策提醒模块联动（如签证政策变化会标红相应节点） |
| **优先级** | 🟡 **P1（重要但可延期）** |

5️⃣ 政策更新与提醒模块

| 项目 | 说明 |
| --- | --- |
| **功能名称** | 智能提醒系统 |
| **功能描述** | 系统根据用户目标国家，实时推送签证、入学、奖学金或政策更新。 |
| **用户流程** | 用户登录 → 主页通知栏显示最新更新 → 点击查看详情 |
| **输入** | 用户目标国家、学校、时间节点 |
| **输出** | 推送通知（弹窗/红点）+ 更新详情页 |
| **状态变化** | 新政策 → 状态标红 → 用户点击后恢复灰色已读 |
| **交互逻辑** | 触发“进度模块”节点状态变化 |
| **优先级** | 🟡 **P1（重要但可延期）** |
